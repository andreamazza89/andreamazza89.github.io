Anyway, something I have been wanting to work on but have not got around to do yet is get some practice on GitHub. I think the best way would be to put together a simple project in Ruby and track it on GitHub.

I got my inspiration for this on Holiday: lots of payments were made by one person for a group of friends, and I kept track of it all to then enter onto a spreadsheet and figure out who owes whom what. How much better would it be if there was an application where you could just enter name of the person, how much they paid and then the program can tell you who owes who and what?!?

Well, I will only be able to make something that runs on IRBÂ and I am not quite sure if I will be able to save the status to reopen and add new entries later, but I guess this is more than good enough for practising on GitHub.

Ok, this is what I figured so far in terms of design. There will be two classes: Person and Receipt.

Each instance of Person will store how much each person has spent and a hash containing how much they owe to the others.

The Receipt class is going to have one instance per trip. Each trip has an array of hashes containing every single expense as follows:

ski_trip_2016 = [{date => date of expense, total_paid => price on receipt, paid_by => person who paid, paid for => who the expense was for, description => quick description }, ...]

Each receipt object will store the total expenditure, as well as update how much a certain person has spent and how much each person owes the others.

Mm reading the above it is a bit confusing but this is the first time I ever think about program design..hopefully I will make more sense of it soon!

In addition to writing the classes above I will also need to write the main program to run a session in IRB and then think about saving and loading existing trips.
