Here we are, week 6 has just finised, which marks the halfway point through the course; I knew it was going too fast!

As you might have read, this week I have been frustrated; there was just about too much we were struggling with within my team and at one point had spent a whole day getting nowhere. What happened is we decided to tackle this week's project using a set of technologies that was completely new to us.

However things started moving from Wednesday and we have learned quite a bit since then. Granted, we are just scraping the surface and are by no means experts in what we learned, but you have to start somewhere, and I am sure learning something this way is much more persistent in the long term than canonical learning, being spoon-fed information.

Anyway, what we spent most time researching before we could start writing the application was a framework to test the whole thing. And yes, before we write any code, we want to have a test for it that tells us what we expect from that code. We tried so many differenent options, and were not even completely sure what we really needed initially.
Zombie was our saviour. Zombie is a NodeJS framework (Node is the main technology we were using) that allows you to have automated tests that do things like:

-> connect to the signup page
-> fill in user information
-> click 'submit'
-> the page you land on should be hostaddress/specific_page_you_should_land_on
-> the page should display the new username in the top section

So instead of running the server, connecting to it and checking features manually we had our friend the Zombie doing these things for us automatically. Thanks Zombie. The first time you send the Zombie, say with the instructions above, it will complain that the signup page does not exist, which makes you create one. Then it will complain there needs to be a signup form on the page, which makes you add one, and so on, driving both page content, navigation and, most importantly, logic.

A funny thing is that Zombie is headless. OK, maybe not that funny; it has been a long week... Headless means it does not need a specific browser to connect to a server, it follows instructions as above and then checks the results. Let me explain further. There are three main ways to test a web application from the point of view of the end user. The obvious one is doing it yourself by manually connecting to the server with a browser and checking it is behaving as expected. This is not very reliable and can become painfully time consuming if we are trying to test several features. So one way to automate is having a webdriver that starts one of your browsers and runs the tests for you (this is oversimplified). This method also allows to check compatibility across multiple browsers/versions of the same browser, however that makes for a much longer setup and overkill for what we were doing. It is pretty cool to see your browser by a third party though, much like Neo being hacked in The Matrix (not really, but sure Andrea..). Finally we have the headless method described above, which is much faster and does not require an actual browser, instead it connects to a server directly and analyses its responses. Cool no?

Final point on Zombies: there is a great song by Fela Kuti called Zombie, check it out.

I am happy to say I have learned plenty this week, despite the initial hiccup. In fact, one of the biggest takeaways is how I should deal with failure and learning completely new things in a short span of time. The former is easier than it sounds: sucking at something is good :) It means that there is room to get better and getting better at anything is a lot of fun!
It was much more difficult for me to learn new things quickly; my main problem with it is not being able to formulate a somewhat thorough understanding of the concept before being able to put it into practice. This slows me down. I am not saying having a shallow understanding of concepts is good, but, especially in programming, when learning completely new concepts, getting your hands dirty soon helps figuring things out and allows you to move on faster. This is related to a concept I am very fond of, information hiding, which I will talk about in the future.
