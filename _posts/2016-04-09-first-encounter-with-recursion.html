It has finally happened. Out of nowhere, while working on the solution to a problem on codewars, I just did it, I used recursion! It was magic.
Recursion is something I had known for a while but always figured I should leave it as something to learn later as it is a fairly advanced topic for someone who has just started.

In computer science, recursion is a process whereby a function invokes itself.
Here is a simple example:

[f(x) = x + f(x + 1)]

Well, although simple, the above is useless, as it creates an infinite loop and never yields a result. Try to substitute x with any number and work out the result to see how the function keeps calling on itself, forever!

So what is the point with recursion? The trick is in defining a function the eventually reaches a stop.

Building on the example above, we could arbitrarily impose that the function will return 0 if x is larger than say 3. Let us try and work out the result for x = 0 then.

First the new definition: [f(x) = x + f(x + 1)], return 0 if x > 3

f(0) = 0 + f(1)
f(1) = 1 + f(2)
f(2) = 2 + f(3)
f(3) = 3 + f(4)
f(4) = 0, since (x > 3)

Building backwards, f(3) is 3, so f(2) is 5 and f(1) is 6, giving us f(0) = 6.

Recursion -when possible- offers a much more elegant solution to its non-recursive counterpart (at least I think, as I am not an expert!). Recursion is a little magic, as it allows an infinite process to be generated from a finite statement. For it to be useful in computing, the infinite process has to stop at some point, but still, pretty neat, no?
